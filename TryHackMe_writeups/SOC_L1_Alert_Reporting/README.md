## SOC L1 Alert Reporting - TryHackMe Room

This repository contains two SOC Level 1 alert reports created as part of hands-on defensive security practice. Each report is based on the simulated alerts from the TryHackMe platform and demonstrates the workflow, analysis, and documentation expected from an L1 analyst in a real Security Operations Centre.

The goal of this repository is to showcase:
- Ability to triage and validate alerts.
- Clear and concise analytical writing.
- Understanding of attacker behaviours and common detection patterns.
- Proper escalation practices within a L1 -> L2 workflow.
- Professional documentation suitable for SOC ticketing systems.

---

## What's Included

Each alert write-up follows the structure below:
- **Summary** - High-level overview of the alert and key indicators.
- **Process Lineage** - Breakdown of source, parent, and grandparent processes, where applicable.
- **Assessment** - L1-appropriate analysis of activity and intent.
- **Escalation Notes** - Clear justification for escalation to L2.
- **Screenshot** - Supporting evidence of the alert from the alert console.
